# Maverick KB — чеклист готовности инструкции

Используй этот чеклист перед тем, как сказать пользователю `готово`.

## 1. Источник обработан

- [ ] Исходная папка/файл существует.
- [ ] Список материалов составлен.
- [ ] Для видео учтены и аудио, и экран.
- [ ] Транскрипт получен или явно зафиксировано, почему он невозможен.
- [ ] Ключевые скриншоты извлечены.
- [ ] Скриншоты обрезаны до полезной области: без панели встречи, таскбара, лишних окон.
- [ ] Из материала выделены подтверждённые факты.
- [ ] Неуверенные факты отделены от подтверждённых.

## 1.1. Анти-сбой транскрипции

- [ ] Агент проверил `D:\\soft\\Whisper\\whisper-cli.exe`, а не только команду `whisper` в PATH.
- [ ] Агент знает Git Bash/MSYS путь `/d/soft/Whisper/whisper-cli.exe`.
- [ ] Агент не требовал PowerShell-команду `whisper`.
- [ ] Агент не требовал `OPENAI_API_KEY` как обязательное условие.
- [ ] Если качественный транскрипт не получился, агент всё равно извлёк визуальные опоры и сделал полезную часть результата.
- [ ] Спорные факты ушли в `QUESTIONS.md`, а не остановили всю задачу.

## 2. Существующая база проверена

- [ ] Поисканы существующие страницы по теме.
- [ ] Каноническая страница определена.
- [ ] Решено: обновить существующую, создать новую, слить дубли или вынести в вопросы.
- [ ] Новая страница не дублирует старую.
- [ ] Противоречия со старым контентом проверены.
- [ ] Неразрешённые противоречия вынесены в `QUESTIONS.md`.

## 3. Страница написана как инструкция

- [ ] Название отражает пользовательскую задачу.
- [ ] Есть понятное вступление: что помогает сделать страница.
- [ ] Указано, для кого инструкция.
- [ ] Указано, когда применяется.
- [ ] Указано, что получится.
- [ ] Есть порядок действий или алгоритм проверки.
- [ ] Есть проверка результата.
- [ ] Есть предупреждения по рискам, если тема рискованная.
- [ ] Есть частые ошибки или диагностика, если они известны.
- [ ] Есть связанные страницы.
- [ ] Пустые разделы удалены.

## 4. Стиль чистый

- [ ] Нет пересказа видео.
- [ ] Нет сырых фрагментов транскрипта.
- [ ] Нет фраз `по видео`, `в источнике`, `как сказала Настя`.
- [ ] Нет `возможно`, `скорее всего` как замены факта.
- [ ] Нет `TODO`, `QUESTIONS`, `high-risk`, `verified` в публичной странице.
- [ ] Нет внутренних комментариев участников.
- [ ] Нет неподтверждённых финансовых, налоговых или бухгалтерских утверждений.
- [ ] Текст написан коротко и понятно.

## 5. Вопросы и пробелы оформлены

- [ ] Все неизвестные факты вынесены в `Maverick_KB/QUESTIONS.md`.
- [ ] Каждый вопрос содержит ссылку на страницу, из которой он возник.
- [ ] Вопрос сформулирован так, чтобы владелец процесса мог ответить.
- [ ] В публичной статье нет фразы `нужно уточнить` вместо содержания.

## 6. Vault обновлён

- [ ] Страница записана в `D:\JetBrains\cl_maverick_kb\Maverick_KB\wiki`.
- [ ] Изображения записаны в `D:\JetBrains\cl_maverick_kb\Maverick_KB\media`.
- [ ] `Maverick_KB/INDEX.md` обновлён, если добавлена новая страница.
- [ ] `Maverick_KB/CHANGELOG.md` обновлён.
- [ ] `Maverick_KB/QUESTIONS.md` обновлён, если остались вопросы.

## 7. Pages синхронизированы

- [ ] Страница скопирована/сконвертирована в `D:\JetBrains\cl_maverick_kb_pages\docs`.
- [ ] Изображения скопированы в `D:\JetBrains\cl_maverick_kb_pages\docs\assets\media`.
- [ ] Obsidian-картинки `![[...]]` заменены на Markdown-картинки.
- [ ] Wikilinks `[[...]]` заменены на Markdown-ссылки.
- [ ] `mkdocs.yml` обновлён, если добавлена новая страница в навигацию.

## 8. Техническая проверка

Из `D:\JetBrains\cl_maverick_kb_pages`:

```bash
uvx --with mkdocs-material mkdocs build --strict
```

- [ ] Build успешен.
- [ ] Битых Markdown-ссылок нет.
- [ ] Отсутствующих картинок нет.
- [ ] В публичных `.md` нет `[[...]]`.
- [ ] В публичных `.md` нет внутренних маркеров.
- [ ] Search index содержит новые/обновлённые страницы.
- [ ] Поиск проверен по реальным пользовательским фразам.

## 9. Визуальная проверка

- [ ] Страница открыта локально или live.
- [ ] Страница выглядит как инструкция, не как конспект.
- [ ] Скриншоты читаемы.
- [ ] Скриншоты поддерживают конкретные шаги.
- [ ] Нет лишних панелей, окон, таскбара.
- [ ] Таблицы и списки читаемы.

## 10. Публикация

- [ ] Изменения в Pages repo добавлены в git.
- [ ] Commit создан.
- [ ] Push выполнен.
- [ ] GitHub Actions deploy завершился `completed success`.
- [ ] Live URL изменённой страницы возвращает HTTP 200.
- [ ] Ключевые изображения возвращают HTTP 200.
- [ ] Live search index обновился.

## 11. Финальный ответ

Финальный ответ должен содержать:

- [ ] что создано/обновлено;
- [ ] где лежит;
- [ ] что проверено;
- [ ] live URL, если была публикация;
- [ ] какие вопросы остались, если есть.

Не писать `сделаю`, `надо будет`, `можно потом`. Если задача требовала результата — он должен быть сделан и проверен.
