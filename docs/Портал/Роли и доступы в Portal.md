# Роли и доступы в Portal

Раздел `Роли` показывает роли Portal, их разрешения и доступные действия с ролью.

<div markdown>

## Где находится

Portal → `управление` → `Роли`.

## Что есть на странице

На странице `РОЛИ` видна таблица с колонками:

- `Имя`;
- `Разрешения`;
- `Действия`.

Доступны кнопки:

- `Добавить`;
- `Изменить`.

В текущем интерфейсе видна роль `ADMIN`. В списке разрешений отображаются страницы и метрики, например:

- `страница главная`;
- `страница бонусы`;
- `страница новости`;
- `страница цифровая реклама`;
- `страница поиск билетов`;
- `страница поиск сертификатов`;
- `страница роли`;
- `страница пользователи`.

## Как проверять роль

1. Открой `Роли`.
2. Найди нужную роль по имени.
3. Проверь список разрешений.
4. Если нужно изменить роль, используй `Изменить`.

## Важно

!!! warning "Доступы управляют возможностями сотрудника"
    Не меняй роль и разрешения без подтверждения владельца доступа. Ошибка в роли может открыть сотруднику лишние операции или закрыть нужный рабочий раздел.

## Раздел Пользователи

В текущем интерфейсе страница `Пользователи` открывается с сообщением `The page is under development ...`. Поля и действия пользователей на этой странице не подтверждены.

## Частые ошибки

- Проверяют доступ по видимости кнопки, но не смотрят список разрешений роли.
- Меняют роль без фиксации, какие разделы должны быть доступны сотруднику.
- Ожидают рабочую страницу пользователей, хотя текущий экран показывает состояние разработки.
