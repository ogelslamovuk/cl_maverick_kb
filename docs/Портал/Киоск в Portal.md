# Обновление баннеров киоска через Portal

Раздел `Киоск` хранит пути и файлы изображений, используемых интерфейсом киоска.

## Где находится

Portal → `управление` → `Киоск`.

![Настройка баннеров киоска в Portal](../assets/media/portal/portal_kiosk_ru.png)

## Поля

| Поле | Назначение |
|---|---|
| `Основной баннер продукта` | основной баннер продукта |
| `Основной баннер билет` | основной баннер билета |
| `Основное изображение комбо` | изображение карточки комбо |
| `Основной баннер комбо` | основной баннер комбо |
| `Стандартный баннер продукта` | изображение по умолчанию для продукта |
| `Стандартный баннер фильма` | изображение по умолчанию для фильма |

У каждого поля есть:

- предпросмотр текущего изображения;
- текстовое значение с путём или ссылкой на изображение;
- выбор файла;
- кнопка `Загрузить`.

Текстовое поле хранит путь вида `images/kiosk/...webp`. Кнопка без подписи рядом с ним открывает выбор файла; `Загрузить` отправляет выбранный файл. Предпросмотр позволяет проверить, к какому баннеру относится строка до изменения.

Форма выбора файла принимает `WEBP`.

## Перед загрузкой

1. Проверь назначение поля.
2. Подготовь файл `.webp`.
3. Убедись, что изменение согласовано с владельцем киоска или контента.
4. После загрузки проверь изображение в клиентском интерфейсе киоска.

## Важно

!!! warning "Изменение влияет на клиентский интерфейс"
    Неверный файл или путь может заменить баннер в киоске. Не загружай изображение без согласования и проверки назначения.

## Частые ошибки

- Загружают файл не в формате `WEBP`.
- Путают изображение карточки комбо и баннер комбо.
- Меняют стандартное изображение вместо баннера конкретного типа.
- Не проверяют результат в киоске после загрузки.
